# Sniper Simulation Modern CLI

This directory contains a modernized Python CLI for the Sniper simulator, providing a more structured and user-friendly interface for running simulations and analyzing results.

## Features

- **Organized Results:** Every simulation run is stored in its own directory under `results/` with metadata, traces, and statistics.
- **YAML Configuration:** Support for `config/sniper.yaml` and `~/.sniper.yaml` for persistent user preferences.
- **Environment Diagnostics:** `sniper-cli.py doctor` checks your environment and dependencies.
- **Rich Output:** Colored tables and progress indicators using the `rich` library.
- **Run Management:** Easily list and inspect previous simulation runs.
- **Integrated Stats:** Direct access to simulation statistics from the CLI.

## Commands

### `doctor`
Check if Sniper and its dependencies (SDE, Pin, XED) are correctly configured.
```bash
python3 sniper-cli.py doctor
```

### `sim`
Perform a full simulation: record a trace and run it through Sniper in one step.
```bash
python3 sniper-cli.py sim --name my_simulation -- /path/to/app args...
```

### `runs list`
List all previous simulation runs stored in the `results/` directory.
```bash
python3 sniper-cli.py runs list
```

### `runs stats`
Display key statistics for a specific simulation run.
```bash
python3 sniper-cli.py runs stats <run_name>
```

### `record`
Record a SIFT trace only.
```bash
python3 sniper-cli.py record -o my_trace -- /path/to/app
```

## Configuration

The CLI looks for configuration in the following order:
1. `config/sniper.py` (Auto-generated by Makefile)
2. `config/sniper.yaml` (Project-level user config)
3. `~/.sniper.yaml` (User-level global config)

Example `sniper.yaml`:
```yaml
sde_home: "/path/to/sde"
pin_home: "/path/to/pin"
target: "intel64"
```

## Implementation Details

The CLI is built using:
- **Click:** For the command-line interface structure.
- **PyYAML:** For configuration management.
- **Rich:** For beautiful terminal output.
- **RunManager:** A new component in `python/sniper/run_manager.py` that handles the `results/` directory structure.
- **SniperConfig:** Enhanced in `python/sniper/config.py` to handle multiple config formats and environment setup.
